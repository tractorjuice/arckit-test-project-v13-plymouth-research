## Summary

In-process analytics databases (like DuckDB) represent a significant evolution from traditional transactional in-process databases (like SQLite). While both run within the application process, eliminating server management, they are optimized for fundamentally different workloads. SQLite excels at transactional (OLTP) workloads with row-based storage, ideal for "point queries" and writes. In-process analytics databases use columnar storage and vectorized query execution, making them exceptionally fast for analytical (OLAP) queries that aggregate large amounts of data.

## Key Findings

- **Optimized for Different Workloads**: SQLite is for OLTP (fast writes, point reads). DuckDB is for OLAP (fast scans and aggregations). They are complementary, not mutually exclusive.

- **Zero-ETL Analytics**: An in-process analytics database like DuckDB can read data directly from SQLite files, Parquet files, or Pandas DataFrames. This allows for high-performance analytics on existing data without a separate ETL process.

- **Columnar Storage**: Data is stored by column, not by row. When calculating an average over a single column (e.g., `AVG(price)`), the engine only reads the `price` column, drastically reducing I/O compared to a row-based system which must read every row.

- **Vectorized Execution**: Queries are processed in batches of values ("vectors") at a time, rather than per-row. This CPU-cache-friendly approach provides orders-of-magnitude performance improvements for analytical functions.

- **Deployment Simplicity**: Like SQLite, these databases are serverless. They are embedded as a library within the application (`pip install duckdb`). This makes them ideal for serverless environments, lightweight applications, and local data analysis.

## Relevance to Projects

- **Plymouth Research Restaurant Menu Analytics (Project 001)**: This technology is highly relevant. The project uses SQLite for its primary data store. The dashboard, however, needs to perform complex analytical queries for its reporting tabs (e.g., average price per cuisine). By adding DuckDB, the dashboard can perform these aggregations directly on the SQLite database file with 10-100x better performance than using Pandas, without needing to migrate the primary database to a more complex system like PostgreSQL. This is a zero-cost, low-effort performance enhancement.
